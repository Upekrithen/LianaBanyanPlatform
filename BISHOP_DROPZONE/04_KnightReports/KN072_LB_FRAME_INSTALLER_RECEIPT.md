## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Installer location | `lb_frame_installer/` inside workspace | Lives with the platform source; version-controlled |
| Package format | `.zip` via `Compress-Archive` | No extra tooling required; native PowerShell 5.1 |
| Version naming | `v0.1.0-walkaround-demo` | Semver + stage label for clear identification |
| Walkaround delivery | Copy to `Documents/` then execute | Matches user's existing workflow (they already have it there) |
| Settings.json merge | Additive only (never destructive) | Preserves user customizations; -Force to re-add if needed |
| Claude Code detection | `Test-Path ~/.claude/` | Not `claude` CLI on PATH — GUI users don't have CLI on PATH |
| Idempotency | `Copy-IfNew` helper + skip-if-exists | Safe to run repeatedly; `-Force` overrides for re-install |
| Tests | Design validation (not live execution) | Running the installer would modify the system; design tests are stable and fast |

## Smoke-Test Note

**Manual smoke-test recommended before end-user distribution.**

The auto-fire Walkaround captures exit code and surfaces any Cathedral issues to the
installer output. Run the full installer on a clean machine / test VM before shipping:

```powershell
powershell -ExecutionPolicy Bypass -File LBFrame-Setup.ps1 -DryRun  # safe preview first
powershell -ExecutionPolicy Bypass -File LBFrame-Setup.ps1           # live install
```

The Walkaround exit code drives the installer's own exit code (0 = fully green, 1 = fix needed).
This means CI/automation can detect incomplete Cathedral installs automatically.
