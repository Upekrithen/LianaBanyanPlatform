### Phase C — Island-spec schema + Supabase migration

7. **Migration `member_islands_table.sql`**:
   - `member_islands` table:
     - `island_id` UUID PRIMARY KEY
     - `member_id` UUID REFERENCES members(id) — creator
     - `name` text — member-authored island name
     - `real_world_twinning_anchor` jsonb — geographic + cultural + structural anchor data
     - `ghost_world_state` jsonb — full island-spec body (mechs / lore / inhabitants / events)
     - `tensor_coordinate` jsonb — `{location, dimension, time_anchor}` per Multi-dim Twinning canon
     - `pedestal_forum_decree_id` UUID — links to `island_pedestal_forum_additions` for member-additions
     - `ip_ledger_stamp_id` UUID — IP Ledger creator-stamp
     - `marks_payout_rate` numeric — initial Marks payout (Tier 6 multiplier × base; recalibrated on canon-class promotion)
     - `cohort_class_eligibility_visit` text[] — which cohort classes can visit (default: all-LB-members; gating via 3-access-key still applies)
     - `access_key_config` jsonb — `{deck_card_durable: bool, guide_mediated: bool, babylon_candle_consumable: bool}` per FLAG_MARKER FM-001
     - `canon_class_status` enum — `pending_review` / `member_class` (default after creation) / `canon_promoted` (Founder Fire Code only)
     - `created_at`, `updated_at` timestamps
   - `island_pedestal_forum_additions` append-only ledger table (extends Bushel 13 `paper_pedestal_forum_additions` pattern):
     - `addition_id` UUID PRIMARY KEY
     - `island_id` UUID REFERENCES member_islands(island_id)
     - `decree_author_member_id` UUID REFERENCES members(id) — Mordecai-Esther co-author
     - `decree_text` text — co-equal-authority addition (CONTRADICTORY-OR-EXTENDING legally-valid decree)
     - `decree_class` enum — `extending` / `contradicting` / `composing` (per Mordecai-Esther canon)
     - `appended_at` timestamp
     - `original_island_immutable_attestation` boolean DEFAULT TRUE — Law of the Medes and Persians: original is NEVER modified; only additions compose alongside
8. **8 RPCs for island-CRUD**:
   - `rpc_create_member_island(spec jsonb, real_world_anchor jsonb, tensor_coord jsonb)` → returns island_id; spawns IP Ledger stamp; emits Pheromone
   - `rpc_get_member_island(island_id)` → full spec + decree count
   - `rpc_list_member_islands(filter jsonb)` → filtered list (by member, by tensor coord, by canon_class_status)
   - `rpc_visit_member_island(island_id, access_key jsonb)` → returns spec IF access-key gating passes; emits visit Pheromone; credits creator with visit-count + Marks
   - `rpc_append_island_decree(island_id, decree_text, decree_class)` → Mordecai-Esther append; original-island-immutable enforced at table level
   - `rpc_promote_island_to_canon(island_id, founder_fire_code_token)` → Founder-only; transitions canon_class_status; canon-promoted islands gain Wrasse anchor + cross-island federation routing
   - `rpc_island_marks_payout(island_id, period)` → calculates accrued Marks for creator (visits × visit-class × Tier 6 multiplier)
   - `rpc_island_provenance(island_id)` → returns full provenance: creator + IP Ledger stamp + decree chain + visit log + canon-promotion history
