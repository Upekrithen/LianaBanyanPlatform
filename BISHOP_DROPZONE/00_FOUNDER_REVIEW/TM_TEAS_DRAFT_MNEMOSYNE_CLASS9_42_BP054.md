## SPECIMEN NOTES — DETAILED GUIDANCE

### Class 9 (downloadable software) — what constitutes a valid specimen

Per TMEP §904.03(e), acceptable specimens for downloadable software include:
- Screenshots showing the mark in the title bar, menu bar, or splash screen of the running application
- Screenshots of the download page at mnemosynec.ai showing the mark adjacent to the download button
- Screenshots of the installer screen showing the mark (NSIS installer, Windows)
- Physical packaging or box art if applicable (not applicable here — digital distribution only)

**Not acceptable:**
- Source code files
- Internal development screenshots not visible to end consumers
- README.md or documentation files alone (without showing the mark in the software itself)

**Best specimen for this application:**
Screenshot of the installed Mnemosyne v0.1.10 application showing:
- Title bar: "Mnemosyne v0.1.10"
- Application header or About screen
- Date of screenshot must be on or after the first use in commerce date declared

The title bar "Mnemosyne v0.1.10" display was implemented in commit `ea37ea5` (KniPr011, 2026-05-23). Take screenshot on or after 2026-05-23 for strongest specimen.

### Class 42 (SaaS/online services) — what constitutes a valid specimen

Per TMEP §1301.04, acceptable specimens for online services include:
- Screenshot of the web application with the mark displayed in the interface
- Printout from the service URL showing the mark and the nature of the services

If the service is not yet live and accessible to public users, **do not attempt to manufacture a specimen** — file 1(b) and submit the specimen when the service launches (Statement of Use or Extension).
