## 5. Heading Hierarchy

**PROV_16 canonical heading system:**
| Level | Used for | Example |
|-------|----------|---------|
| H1 (`#`) | Document title only | `# Provisional Patent Application 16` |
| H2 (`##`) | Major structural sections | `## Filing Manifest`, `## Background of the Invention`, `## Claims`, `## Abstract`, `## Filing Gate Status` |
| H3 (`###`) | Innovation sections, problem subsections | `### Section I — Innovation Title (#NNNN)`, `### The Memory Cost Explosion Problem` |
| H4 (`####`) | Sub-subsections within innovation | `#### Background and Problem Statement`, `#### Detailed Description` |

**Heading casing:** Sentence case (capitalize first word + proper nouns only).  
NOT ALL CAPS.

**v05_FINAL issues:**
- `## BACKGROUND OF THE INVENTION` → `## Background of the Invention`
- `## SUMMARY OF THE INVENTION` → `## Summary of the Invention`

- `## EXTENDED INNOVATIONS (BP084: CLAIM GROUPS 28-30)` → normalize as a comment or remove divider
- `## BRIEF DESCRIPTION OF DRAWINGS` → `## Brief Description of Drawings`
- `## ABSTRACT` → `## Abstract`
- `### Field of the Invention` (H3 inside Background) → `## Field of the Invention` per PROV_16 (H2 standalone section OR keep H3 — acceptable)
- `## Filing Gate Status` — already sentence case ✓

## 10. Claim Notation

**PROV_16 canonical:**
```
**Claim N** (Innovation Cluster Name — Independent/Dependent Method/System Claim)

A method for..., comprising:
- (a) first element;
- (b) second element;
- (c) third element.

**Claim N+1** (Innovation Cluster Name — Dependent: Specific Property)

The method of Claim N, wherein...
```

**Key features:**
- `**Claim N**` with "Claim" keyword — NOT bare `**N**` or `**N.M**`
- Title label in parentheses: "(Cluster Name — Type)"
- Sub-clauses as **bullet list**: `- (a)`, `- (b)`, `- (c)` (NOT inline prose)
- New paragraph between claim header and body

**v05_FINAL current notation:**
- `**N.M**` — no "Claim" keyword, no title label
- Sub-clauses: `(a) text` on new line but NOT bulleted
- Example: `**1.1** A computer-implemented method...`

**v06 target:**
- `**Claim 1.1** (Canonical Plow Pipeline — Independent Method Claim)` 
- Keep CG.X numbering for navigability across 37 CGs (sequential global numbering ~185+ claims is unwieldy)
- Convert sub-clause lines to `- (a)` bullet format

## 13. PDF Render Toolchain (per build_prov16_pdf.py)

**Pandoc command:**
```
pandoc [input.md] -o [output.html]
  --standalone
  --toc --toc-depth=2          ← CRITICAL for TOC in PDF
  --metadata title=[TITLE]
  -V geometry:margin=1in
  --from=markdown+raw_tex
  --to=html5
```

**CSS for page formatting:**
- Line height: 1.38
- Font: Times New Roman 12pt
- Page size: Letter
- Margins: 1in all sides

**Chrome headless command:**
```
chrome.exe --headless=new --disable-gpu --no-sandbox
  --print-to-pdf=[output.pdf]
  --print-to-pdf-no-header
  [input.html]
```

**v05 render spec state:** CSS LH 1.38, 12pt TNR, Letter, 1in margins — CSS already correct.  
**Missing:** `--toc --toc-depth=2` flag → THIS is why v05 PDF has no TOC.  
**Fix:** Add `--toc --toc-depth=2` to pandoc command for v06.
