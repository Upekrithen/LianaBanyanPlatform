## 10. Complete TEAS Plus Filing Walkthrough — Step-by-Step for Founder Self-Filing

**Before you begin:** Have this document open alongside teas.uspto.gov. Allow 45–60 minutes for a first filing. All five applications follow the same portal structure.

### Step 1 — Log into USPTO / Create Account
1. Navigate to https://teas.uspto.gov/initial/
2. Click **"TEAS Plus"** (not TEAS Standard — TEAS Plus is $250/class, Standard is $350/class)
3. Sign in with your USPTO.gov account (or create one — use a long-lived corporate email)
4. Select **"Start a new TEAS Plus application"**

### Step 2 — Mark Information
1. **"Is the mark in standard characters?"** → Select **YES** (for CAITHEDRAL word mark)
2. **"Enter the mark"** → Type: `CAITHEDRAL` (all caps, as filed)
3. **"Does the mark include non-Latin characters or special characters?"** → NO
4. **"Translation / Transliteration"** → Leave blank (coined word, no translation)
5. **"Disclaimer"** → Leave blank (fanciful coined word; no disclaimer needed)

### Step 3 — Goods and Services (Class 9)
1. Click **"Add Goods/Services"**
2. Select **Class 009** from the dropdown
3. In the ID Manual search box, search and select each entry by code:
   - Type `009-2199` → select "Downloadable computer software for artificial intelligence (AI)"
   - Type `009-1638` → select "Downloadable computer software for machine learning"
   - Type `009-1612` → select "Downloadable computer software for managing and organizing personal data"
   - Type `009-3127` → select "Downloadable computer software for knowledge management"
   - Type `009-0395` → select "Downloadable computer software for use as an application programming interface (API)"
4. Click **"Accept"** — verify total reads $250 (not $450); if $450, a free-text entry triggered the surcharge; delete and re-select from ID Manual

### Step 4 — Filing Basis
1. Select **"Section 1(b) — Intent to Use"** (recommended) OR **"Section 1(a) — Use in Commerce"** if accepting CaithedralInspector.tsx or Cephas Hugo as specimen
2. If 1(a): enter dates:
   - **"Date of first use anywhere"**: `2026-05-23`
   - **"Date of first use in commerce"**: `2026-05-23`
