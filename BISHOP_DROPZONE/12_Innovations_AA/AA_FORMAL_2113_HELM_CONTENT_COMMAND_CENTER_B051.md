## #2113 — Helm Content Command Center
**Session**: K190
**Category**: Infrastructure / Admin
**Crown Jewel**: No
**Prior Art**: CMS dashboards exist everywhere. But a Founder-facing pre-launch review tool that seeds ~300 outbound documents (letters, articles, papers, pitches, pudding essays, cue cards) from multiple source tables into a single review queue with keyboard-navigated approve/reject workflow, markdown preview, wave assignment, and Plugs/Cue Card integration — that's ours.

### Description
Founder-facing content management dashboard for reviewing and approving ALL ~300 outbound documents before launch. Database: `helm_content_queue` table with 17 content types, 7 status values, full review workflow fields. Seeded from `cephas_content_registry` plus explicit inserts for all letter types and pudding essays. UI: 4-stat header, 5 filters + search + sort, list/grid toggle, full markdown reader with review controls sidebar, keyboard shortcuts (A/R/S/arrows/Esc). Integration with Plugs dispatch and Cue Card generator.

### Innovation Markers
- 17 content types unified in single review queue
- Keyboard-driven review workflow (approve/reject/skip with nav)
- Auto-seeded from multiple source tables (cephas_content_registry, explicit letter/essay inserts)
- Wave assignment for staged outbound dispatch
- Integrated with existing dispatch and cue card systems

### Formal Claim
A cooperative platform content review system that aggregates outbound documents from multiple source registries into a unified review queue with type-specific categorization, priority ordering, and wave-based dispatch scheduling, wherein the review interface provides keyboard-driven navigation with single-keystroke approve, reject, and skip actions, markdown-rendered document preview, and direct integration with social media dispatch and credential-sharing subsystems.
