## TL;DR

A universal momentum metric that normalizes all cooperative platform progress thresholds into a single 0-100 scale called Catapult Power (CP). The metric is analogous to horsepower for engines: users do not need to understand the underlying units of any specific threshold (dollars pledged, signatures gathered, votes counted, members enrolled) because the CP score maps any threshold-based accumulation to the same universal scale. **100 CP equals escape velocity — the point at which the tracked entity achieves its activation threshold and launches.** The metric is applied uniformly across project backing, political petition signature collection, legislative vote gathering, kickstarter funding, initiative launch, and submissions pedestal voting, enabling cross-system comparability without unit-specific cognitive load on the cooperative member.

## Independent Claim

**Claim 1.** A computer-implemented method for normalizing heterogeneous progress thresholds across multiple entity types in a cooperative platform into a universal momentum metric, comprising:

(a) Maintaining a catapult metrics database wherein each record comprises an entity type classification from a set of supported entity types, an entity identifier, a current progress value, a target threshold value, and a generated universal momentum score computed as the minimum of (current / target × 100) and 100;

(b) Exposing a reusable visual gauge component that renders the universal momentum score as a 0-to-100 arc visualization with an amber-to-gold gradient fill, suitable for embedding in any cooperative platform context (dashboard, campaign page, submission page, petition page);

(c) Surfacing the universal momentum score consistently across all entity types through a single query interface, such that cooperative members may compare the momentum of heterogeneous entities (project funding vs. petition signatures vs. campaign votes) on the same scale;

(d) Triggering a launch event when any entity's universal momentum score reaches 100, wherein the launch event initiates entity-type-specific downstream actions (release of held funds, activation of campaign, approval of initiative).

**Dependent Claim 1.1** — The method of Claim 1, wherein the supported entity types include project, petition, vote, campaign, initiative, and submission, and the entity type set is extensible via database constraint modification without requiring schema restructuring.

**Dependent Claim 1.2** — The method of Claim 1, wherein the universal momentum metric is a display-only value during accumulation and does not itself carry currency or economic weight; launch events at 100 trigger downstream economic flows in the cooperative's three-currency system (Credits, Marks, Joules).

**Dependent Claim 1.3** — The method of Claim 1, wherein the universal momentum score is stored as a generated database column, enabling index-based query optimization for dashboard display sorted by nearest-to-launch momentum.

**Dependent Claim 1.4** — The method of Claim 1, wherein the visual gauge of (b) implements a catapult metaphor such that the 0-to-100 arc represents the pulling back of a catapult arm and reaching 100 represents the release of the arm, providing intuitive cross-system progress communication independent of the underlying unit being tracked.

**Dependent Claim 1.5** — A system comprising a processor, a persistent catapult metrics database, a visual gauge component, and instructions stored on a non-transitory computer-readable medium which, when executed, cause the processor to perform the method of Claim 1 through Claim 1.4.

## Prior Art Distinction

Progress bars and threshold displays are universal in software (Kickstarter funding progress, Change.org petition signature counts, etc.), but no existing platform normalizes heterogeneous threshold types into a single universal metric computed from the same generated-column formula and displayed through a single visual component. **The Catapult Power metric creates cross-system comparability that allows a member to evaluate the relative momentum of a funding campaign, a political petition, a submissions pedestal vote, and a cooperative initiative without knowing the underlying units.** The universal-metric approach is novel.
