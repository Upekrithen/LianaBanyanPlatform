## Innovation #2005 — Cephas Community-Curated Resource Linking: Member and User-Suggested Cross-References with Voting

### Description

A system and method within a cooperative commerce platform's knowledge base (Cephas) wherein:

(a) Any user (member OR non-member) can suggest a link between a Cephas documentation page and another resource — another Cephas page, an external article, a Vault document, a platform feature page, or any relevant URL. The suggestion includes the target resource and a brief applicability statement.

(b) Members vote UP or DOWN on suggested links based on how applicable and helpful the link is, using the platform's existing Document Voting rules:
  - One vote per link per member (no spam)
  - All votes public (transparency, no anonymous gaming)
  - Votes locked once cast (commitment)
  - Reputation-only (no Credits or Joules spent — low friction)

(c) Links rise and fall based on community consensus:
  - **Net positive** (upvotes > downvotes): Link remains visible
  - **+10 net votes**: "Community Verified" status (prominent display)
  - **+50 net votes**: "Canonical Link" — pinned, requires -25 net to unpin
  - **-5 net votes**: Auto-hidden (not deleted, can recover)
  - **Decay**: Links lose 1 vote per 90 days of inactivity (stale links fade)
  - **No time limit**: Voting stays open indefinitely

(d) Captains and administrators retain editorial authority to:
  - Pin links without vote threshold (curated recommendations)
  - Remove spam or inappropriate link suggestions
  - Set maximum visible links per page (5-10 default, prevents clutter)

(e) This transforms Cephas from a static knowledge base into a community-curated, living reference network — where the documentation improves through member participation and the best resources rise to the top organically.

### Integration with Existing Systems

- **Document Voting (Hall of Records)**: Uses the same reputation-only, one-vote-per-document pattern
- **SWOOP threshold pattern**: Net vote thresholds trigger status changes (like initiative activation)
- **Ghost Rules**: Non-member link suggestions persist 24 hours; member suggestions persist permanently
- **Beacon Points**: Suggesting a link that reaches "Community Verified" earns Beacon Points
- **Cephas Hugo**: Links stored in a sidecar data file per page, rendered in the Hugo template

### Patent Relevance: MEDIUM

A cooperative knowledge base system wherein documentation pages accept community-suggested cross-reference links that are voted on by platform members using a reputation-only voting system, with tiered visibility thresholds (visible → verified → canonical → hidden) and natural decay — creating a self-curating documentation network owned by its member-users — represents a novel approach to cooperative knowledge management.

## Innovation #2006 — Bidirectional Knowledge Graph: Cephas Pages as Nodes in a Platform-Wide Reference Network

### Description

A system and method within a cooperative commerce platform wherein:

(a) Every Cephas page becomes a node in a bidirectional knowledge graph. When Page A links to Page B, Page B automatically shows a "Referenced by" section listing Page A. This creates a web of connections that grows organically through both editorial curation and community suggestions.

(b) The knowledge graph extends beyond Cephas itself:
  - Cephas pages can link to platform feature pages (/campaigns, /dashboard, /contingency-operators)
  - Platform feature pages can link back to relevant Cephas documentation
  - Vault documents (Asteroid-Proof Vault) can be referenced from Cephas and vice versa
  - External resources (academic papers, legal references) can be linked with community voting

(c) The graph enables discovery:
  - "Related pages" are computed from the link graph (pages that share many links in common)
  - Beacon Run routes can be generated from the knowledge graph (gamified documentation exploration)
  - The Librarian (AI) uses the graph to answer member questions with relevant Cephas references

(d) Management is distributed:
  - Members suggest links (bottom-up curation)
  - Captains curate their domain's links (middle-tier editorial)
  - Governance votes on "canonical" link status (top-tier community consensus)
  - AI suggests potential links based on content similarity (supplementary)

### Patent Relevance: MEDIUM

A cooperative knowledge platform wherein documentation pages form nodes in a bidirectional knowledge graph with community-curated edges, enabling organic discovery, gamified exploration (Beacon Run through documentation), and AI-assisted reference — with distributed management across members, Captains, and governance — represents a novel application of knowledge graph architecture to cooperative platform documentation.
