# Innovation #1922: Lemon Lot (Peer-to-Peer Vehicle Sharing)
## Patent Relevance: HIGH (Crown Jewel)

### Classification
| Field | Value |
|-------|-------|
| Innovation # | 1922 |
| Name | Lemon Lot |
| Parent Initiative | Rally Group (#9) |
| Session Originated | Bishop 022A |
| Status | DEPLOYED (K85) |

### Description
Peer-to-peer vehicle sharing marketplace named after army base used-car lots. Members list personal vehicles for other members to rent. LB provides matchmaking, scheduling, reputation, and payment processing. LB does NOT provide insurance, set prices, or guarantee condition. Fills the TURO gap (no vehicle age restrictions, no seat count limits, Cost+20% vs TURO's 25-40%).

### Patentable Elements
1. **Cooperative vehicle-sharing marketplace with constitutional margin lock** — Cost+20% permanently fixed, distinguishing from extractive platforms that increase fees over time
2. **Graduated photo documentation protocol** — risk-scaled photo requirements tied to relationship familiarity (mandatory for new partners, trust-based for repeat)
3. **Calendar Plug integration for vehicle availability** — cooperative calendar system with Rally Group Plug manages vehicle scheduling

### SEC/Legal Notes
- Vehicle sharing is a service marketplace, not a financial instrument
- No passive income — vehicle owner must actively manage listing and availability
- LB's marketplace-only role (no insurance, no price-setting) limits liability exposure
- Cost+20% applies to vehicle owner's set price, not to a platform-determined rate

### Description
Recurring commute path matching. Members register daily routes; LB Calendar matches overlapping routes within configurable radius. Person-to-person arrangement, own insurance. Payment optional through LB (Cost+20%) or direct.

### Patentable Elements
1. **Cooperative commute matching with graduated trust mechanics** — first-ride mandatory photo documentation, trust-based relaxation for repeat partners

- LB is route matcher + scheduler, NOT a transportation network company (TNC)
- Members carry own insurance, set own terms
- Clear disclaimer required in every Rideshare Routes interface
- No TNC licensing required because LB doesn't dispatch, price, or insure

# Innovation #1924: Vehicle Contribution Onboarding
## Patent Relevance: HIGH

### Classification
| Field | Value |
|-------|-------|
| Innovation # | 1924 |
| Name | Vehicle Contribution Onboarding |
| Parent Initiative | Rally Group (#9) / Local Wheels (#1915) |
| Session Originated | Bishop 022A |
| Status | Designed, first test case identified (Founder's Suburban) |

### Description
Three-path structured onboarding for members contributing vehicles: Sale (outright to LB), Earn-Down (20% of earnings toward payoff), and Lease-In (future — insurance too complex for v1). Includes asset declaration, repair bounty via Crew Call, valuation at Cost+20%, and fleet assignment.

### Patentable Elements
1. **Three-path cooperative vehicle contribution model** — Sale, Earn-Down, and Lease-In pathways with distinct ownership, insurance, and payment structures
2. **Earn-Down cooperative vehicle acquisition** — 20% of delivery earnings applied to vehicle payoff, converting gig work into asset ownership within cooperative framework
3. **Repair-bounty-to-fleet pipeline** — vehicle needing repair generates Shade-Tree Mechanic Crew Call, repair completion triggers fleet assignment, creating jobs from the onboarding process itself

### SEC/Legal Notes
- Vehicle contribution is a service/asset transaction, not an investment
- Earn-Down is a payment plan toward vehicle ownership, not a profit-sharing arrangement
- Clear title transfer documentation required for Sale path
- Lease-In (future) will require commercial insurance rider analysis

# Innovation #1926: Graduated Photo Documentation Protocol
## Patent Relevance: MEDIUM

### Classification
| Field | Value |
|-------|-------|
| Innovation # | 1926 |
| Name | Graduated Photo Documentation (Safety Ledger) |
| Parent Initiative | Cross-cutting (all transport + Commerce Engine) |
| Session Originated | Bishop 022A |
| Status | DEPLOYED (K85 — Safety Ledger) |

### Description
Photo documentation requirements scale with risk and relationship familiarity. Deliveries: mandatory every time. Rentals: mandatory every time. First ride with new partner: mandatory. Repeat rides: trust reputation. Incidents: retroactive documentation triggered by claim. Photos are timestamped, GPS-tagged, stored 90 days minimum.

### Patentable Elements
1. **Risk-graduated documentation system** — photo requirements scale with transaction risk and relationship maturity, reducing friction for trusted relationships while maintaining evidence for new ones
2. **Configurable trust threshold** — first-ride documentation is mandatory and cannot be disabled; subsequent-ride documentation is optional but member-configurable
3. **Claim-triggered retroactive documentation** — incident claims activate documentation requirements for both parties, creating evidence chains on-demand

### SEC/Legal Notes
- Photo documentation is an operational trust mechanism, not a financial control
- GPS tagging requires disclosure in privacy policy
- 90-day retention must comply with applicable data retention laws
- Member consent for photo storage required at onboarding
