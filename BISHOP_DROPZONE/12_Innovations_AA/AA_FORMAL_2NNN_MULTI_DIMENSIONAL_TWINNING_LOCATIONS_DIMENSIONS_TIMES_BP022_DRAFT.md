## Section 1 — Field of Invention

This invention relates to cooperative-platform lore-space architectures for cooperative member-creator-class digital/physical hybrid environments. More specifically, it relates to a multi-dimensional addressing system where each point in a cooperative-platform world-state is addressable as a coordinate triple (location, dimension, time), with conditional-operator gating, time-axis resolution via a canonical-ledger primitive, and tiered-access-key enforcement of pocket-state visibility/reachability.

## Section 2 — Background / Prior Art

Prior art in this field comprises:

- **Digital twin** systems (industrial IoT class) — typically a binary pair: a physical asset and its digital representation. No lore-space tensor; no time-axis canonical ledger; no member-creator-class composition.
- **Massively-multiplayer-online world-state systems** (MMO class) — typically present-tense only; historical states are not first-class queryable coordinates. World-states are operator-owned, not member-created with cooperative-license + IP-Ledger credit-where-due.
- **Augmented/mixed reality co-presence systems** — typically a single overlay layer (the "AR slice") onto Real World; do not generalize to N orthogonal dimensional slices nor to time-pocket access-control.
- **Time-pocket / pocket-dimension fictional systems** (e.g., Ransom Riggs *Miss Peregrine's Home for Peculiar Children*; Neil Gaiman *Stardust* Babylon Candle) — literary precedent for the access-key gating pattern; the present invention applies the structural pattern to a cooperative-platform substrate with canonical-ledger enforcement.
- **Cooperative-platform digital-twinning** — Decisions_61 (Bishop Cathedral substrate anchor, M:6 decay 5.65) establishes the prior Ghost World / Real World binary Twinning canon; the present invention preserves that binary as the dimension axis and expands to a 3-axis tensor.

The novel combination is a **3-axis lore-space tensor** (locations × dimensions × times) coupled with conditional-operator gating, Chronos-resolved time-axis ledger, and a 3-tier access-key system enforcing pocket-state reachability — operating under cooperative-license + member-cooperative-ownership + Mordecai-Esther co-equal-authority decree-composition.

## Section 3 — Summary of the Invention

A method and system for a cooperative-platform multi-dimensional lore-space architecture where:

(i) every point in the platform's lore-space is addressable as a **coordinate triple** `(location, dimension, time)`;
(ii) **Locations** form the spatial axis (canonical seed-locations + member-created-locations via Mordecai-Esther co-equal-authority decree-composition);
(iii) **Dimensions** form the layer axis with at least five orthogonal values (Real World / Ghost World / Shade / Excalibur / Wrasse) — extending the prior binary-Twinning canon;
(iv) **Times** form the temporal axis with at least five canonical values (Present / Memory / Prospect / Ur-time / Jubilee) — resolved by a canonical-ledger primitive (Chronos);
(v) **Conditional Operators** function as lore-primitive gates determining which dimensional slice is currently active for any given query;
(vi) **Pockets of Time-Space** form persistent canonical-state coordinates within the tensor that require access-key authorization to enter (pattern reference: Miss Peregrine's pocket-dimensions);
(vii) **3 access keys** (Deck Card / Guide / Babylon Candle) provide tiered access classes (membership-tier / social-tier / consumable-tier) to those Pockets;
(viii) **BC / AD calendar convention** governs the time-axis canonical labeling cross-platform.

The architecture composes with the prior #2277 Conductor primitive (as Conditional Operator substrate), the prior #2308 Year of Jubilee Ledger (as Chronos canonical ledger), the prior #2233 Deck Card primitive (extended to dimensional-access-key class), the Member-Island-Creation Bounty (Bushel 31, productization), and the Mordecai-Esther Pedestal Forum (member-island-as-decree-composition addition).

## Section 4 — Detailed Description

### 4.1 The 3-axis tensor

The fundamental structural unit of the invention is a 3-axis tensor over the cooperative-platform lore-space:

```
LoreSpace = Locations × Dimensions × Times
```

Each fully-specified point is a coordinate triple `(L, D, T)` where `L ∈ Locations`, `D ∈ Dimensions`, `T ∈ Times`.

### 4.2 Axis 1 — Locations (the WHERE axis)

Locations are spatial units of lore-space. In the disclosed embodiment (HexIsle), Locations are islands. The Locations set is decomposed into:

| Location class | Examples |
|---|---|
| Canonical seed-Locations | Harvest, Navigate, Engineer, Battle, Seek, Magic, Train (the canonical 7 Islands) |
| Member-created Locations | Any member-island added per Mordecai-Esther co-equal-authority decree-composition (Bushel 31 Member-Island-Creation Bounty) |
| Federation Locations | LB Frame compute nodes serving as location-anchors |
| Null Location | Pre-island void (before world manifestation) |

Canonical seed-Locations are immutable per the substrate-as-immutable-backup canon. Member-created Locations compose alongside per the Mordecai-Esther Pedestal Forum decree-composition pattern. Founder Fire Code authority is required for any member-Location promotion to canonical-class.

### 4.3 Axis 2 — Dimensions (the WHAT axis)

Dimensions are orthogonal layers of reality. The disclosed embodiment specifies at least five Dimension values:

| Dimension value | Description |
|---|---|
| **Real World** | Physical anchor — the actual object/place/person/craft that the Location connects to |
| **Ghost World** | Digital substrate — the CAI-enabled digital representation; runs on cooperative compute |
| **Shade** | The shadow-state between Real and Ghost (transition layer; Chronos-gated) |
| **Excalibur** | The latent-pattern layer — the not-yet-manifested form; substrate potential |
| **Wrasse** | The routing-memory layer — the substrate's memory of prior dimensional visits |

The prior binary Ghost World / Real World Twinning (Decisions_61, M:6 decay 5.65) is preserved as the {Real World, Ghost World} pair on this axis — two of the five Dimension values.

### 4.4 Axis 3 — Times (the WHEN axis)

Times are temporal layers resolved by Chronos. The disclosed embodiment specifies at least five Time classes:

| Time class | Description |
|---|---|
| **Present** | Current Chronos tick — the active substrate state |
| **Memory** | Past state preserved in Iron Tablet / Pheromone substrate |
| **Prospect** | Future projected state (Conditional Operators + Fates routing) |
| **Ur-time** | The primordial state — before any member action; canonical-seed state |
| **Jubilee** | The Year of Jubilee reset state — full substrate reconciliation point |

Calendar labeling on Real-World-anchored Time coordinates uses **BC / AD** convention exclusively (Founder-mandatory cross-platform).

### 4.5 Conditional Operators (extending #2277 Conductor)

Conditional Operators are lore primitives that gate which dimensional slice is active for any given query. Examples:

| Operator | Function |
|---|---|
| `IF dimension = Ghost World AND location.member_owned` | Member's CAI interface is active |
| `IF dimension = Real World AND Chronos.tick = harvest` | Physical harvest mechanism activates |
| `IF time = Memory AND dimension = Wrasse` | Pheromone routing serves cached route |
| `IF location = member_island AND dimension = Excalibur` | Member's latent island pattern is being authored |
| `IF time = Jubilee AND dimension = Ghost World` | Full substrate reconciliation fires |

Conditional Operators are the productized lore-primitive form of the #2277 Conductor's Baton vendor-neutral adaptive routing — extended from model-routing into dimensional-slice-gating. They are NOT code; they are canonical-substrate lore declarations enforced mechanically by Pheromone substrate + Fates routing + Iron Tablet snapshots.

### 4.6 Chronos (extending #2308 Year of Jubilee Ledger)

Chronos is the time-axis resolution system. It extends the prior #2308 Year-of-Jubilee canonical-ledger primitive into a sub-second tick-progression substrate:

- Chronos tick = canonical heartbeat (Ouralis tidal cycle in-game; Librarian rebuild interval substrate-level)
- Each Iron Tablet entry carries a Chronos timestamp
- Pheromone events carry `ts` = Chronos-resolved timestamp
- Fates routing queries by `(time = Present)` or `(time = Memory, decay_threshold = X)`
- Year-of-Jubilee reset corresponds to `(time = Jubilee)` ledger reconciliation

Without Chronos, the (location, dimension) pair reduces back to the prior binary Twinning. Chronos adds the third axis that makes the time-pocket structure queryable.

### 4.7 Pockets of Time-Space (Miss Peregrine pattern)

A Pocket of Time-Space is a coordinate `(L, D, T)` where T is a non-Present Time class — typically a specific historical year (e.g., `(XO Island, Real World, 1836-AD)`), an Ur-time canonical seed, or a Prospect-class projected future.

Pockets persist within the substrate but require an access key to enter. The pattern reference is *Miss Peregrine's Home for Peculiar Children* (Ransom Riggs novel; 2016 film) — pocket dimensions of time/place that the public cannot accidentally walk into. The structural import: the substrate-as-immutable-backup canon requires write-access protection on historical states, and the access-key system provides the read-access protection symmetric to it.

### 4.8 The 3 Access Keys (Deck Card / Guide / Babylon Candle)

Three orthogonal access mechanisms enforce Pocket reachability across three structurally-distinct entry classes:

| Key | Acquisition | Use class | Composes with |
|---|---|---|---|
| **Deck Card** | Member-credential (legit users have these by default) | Persistent / durable / repeatable access | #2233 Deck Card system + Federation membership |
| **Guide** | Social-trust-credential (must be brought through by an existing credentialed member) | Mediated / social / per-visit access | Stitchpunk Corps guide-class + tribe/family/guild relationships |
| **Babylon Candle** | Single-use consumable (Stardust 2007 film reference — single-light teleport candle that takes the holder to wherever they are thinking of when lit) | Transient / one-shot / consumable access | Marks-class consumable cluster + IP Ledger credit-where-due |

**Deck Card** = membership-tier access (the substrate-intrinsic key).
**Guide** = social-tier access (the trust-graph key).
**Babylon Candle** = consumable-tier access (the marketplace key).

Each Babylon Candle ignition logs one entry-event via Channel 4 (Knight → Bishop callback), emits a Pheromone event for cooperative-datacenter compute provisioning, and creates an IP-Ledger credit-where-due stamp paying the candle-creator under the Marks-backed share-back-pay structural form.

### 4.10 Composition with member-island creation (Bushel 31)

When a member creates an island via the Member-Island-Creation Bounty (Bushel 31, productization vehicle), the member is staking a **Location coordinate** that persists across the full tensor — addressable at `(member_island_id, *, *)` for any combination of Dimension and Time values. Other members visit the island at `(member_island_id, Ghost World, Present)` via Wrasse routing; historical visits are recorded at `(member_island_id, Ghost World, Memory)`; future-projection-class authoring lives at `(member_island_id, Excalibur, Prospect)`; founding stamp lives at `(member_island_id, Real World, Ur-time)` permanent in the IP Ledger.

**Claim 1 — Multi-dimensional Lore-Space Tensor**
A computer-implemented method for cooperative-platform lore-space architecture comprising: addressing each point in a cooperative-platform world-state as a coordinate triple `(location, dimension, time)` over a 3-axis tensor where the Locations axis carries at least canonical-seed-locations + member-created-locations, the Dimensions axis carries at least five orthogonal layer values including Real World, Ghost World, Shade, Excalibur, and Wrasse, and the Times axis carries at least five canonical Time classes including Present, Memory, Prospect, Ur-time, and Jubilee — such that the prior binary Twinning canon (Real World ↔ Ghost World) is preserved as the dimension axis of the disclosed tensor.

**Claim 2 — Conditional Operator Gating**
The method of Claim 1 further comprising: a Conditional Operator substrate (extending the prior #2277 Conductor's Baton vendor-neutral adaptive routing) that gates which dimensional slice of the tensor is active for any given query, where Conditional Operators are canonical-substrate lore declarations enforced mechanically by Pheromone substrate, Fates routing, and Iron Tablet snapshots — and where at least one Conditional Operator combines location-class membership-status, dimension-value, and Chronos-time-class as gating predicates.

**Claim 3 — Chronos Time-Axis Ledger**
The method of Claim 1 further comprising: a Chronos canonical-ledger primitive (extending the prior #2308 Year of Jubilee Ledger) that resolves the Times axis via canonical sub-second tick progression, attaches a Chronos timestamp to each Iron Tablet entry and Pheromone event, supports queries by `(time = Present)` and `(time = Memory, decay_threshold = X)`, and reconciles to canonical Year-of-Jubilee reset state at `(time = Jubilee)` — where calendar labeling on Real-World-anchored Time coordinates uses BC / AD convention exclusively.

**Claim 4 — Pockets of Time-Space + 3 Access Keys**
The method of Claim 1 further comprising: persistent canonical-state coordinates `(L, D, T)` where T is a non-Present Time class, designated Pockets of Time-Space, accessible only via one of three orthogonal access keys: (a) a Deck Card membership-tier key extending the prior #2233 Deck Card primitive to dimensional-access-key class; (b) a Guide social-tier key requiring an existing credentialed member to bring the visitor through; (c) a Babylon Candle consumable-tier key consumed in a single-use ignition event that logs an entry-event via Channel 4 callback, emits a cooperative-datacenter compute-provisioning Pheromone event, and creates an IP-Ledger credit-where-due stamp paying the candle-creator under the Marks-backed share-back-pay structural form.

**Claim 5 — Member-Created Location Composition (Mordecai-Esther)**
The method of Claim 1 further comprising: a member-Location-creation pathway under which member-created Locations compose alongside canonical-seed Locations per the Mordecai-Esther co-equal-authority decree-composition pattern; canonical seed-Locations remain immutable per the substrate-as-immutable-backup canon; Founder Fire Code authority is required for any member-Location promotion to canonical-class; and each member-Location stakes a coordinate that persists across the full tensor (location-axis-fixed, dimension-axis-free, time-axis-free).

**Claim 6 — Cross-Tensor Routing**
The method of Claim 1 further comprising: a Federation-Memory-Iceberg cross-tensor routing layer where any member-credentialed visitor at `(L_a, D, T)` can route to `(L_b, D', T')` via Wrasse-anchor lookup at sub-millisecond latency, subject to (i) Conditional Operator gating per Claim 2 and (ii) access-key authorization per Claim 4 for Pocket-class destinations.

**Claim 7 — System Embodiment**
A system implementing the method of any preceding claim, comprising: a cooperative-platform compute substrate (LB Frame), a canonical-substrate Eblet repository with Wrasse-trigger routing, a Pheromone event-bus, an Iron Tablet append-only ledger, a Conductor adaptive-routing module, a Chronos canonical-tick generator, an access-key registry holding Deck Card credentials and Guide social-trust edges and Babylon Candle issued-consumables, and a Mordecai-Esther Pedestal-Forum-additions ledger — operating under the #2260 Cooperative Defensive Patent Pledge umbrella.

**Claim 8 — Vendor-Fungibility**
The method of any preceding claim, where the substrate architecture (Eblets, Scribes, Pheromone, Iron Tablets, Chronos ticks, Conditional Operators) is AI-vendor-agnostic: the lore-space tensor persists when the AI model changes; vendor substitution does not alter the canonical Locations / Dimensions / Times schema (per AI-Agnostic Platform Principle).
