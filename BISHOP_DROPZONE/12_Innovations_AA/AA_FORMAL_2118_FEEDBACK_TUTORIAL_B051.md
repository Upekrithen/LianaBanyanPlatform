## #2118 — Interactive Feedback Tutorial with Sequential Tracking
**Category**: User Experience / Community
**Crown Jewel**: YES — No prior art for a cooperative platform that teaches feedback-as-civic-duty on first visit through an interactive overlay with sequentially numbered submissions that make each contribution feel permanent and tracked.
**Prior Art**: Product tours exist (Shepherd.js, Intro.js, Appcues). Feedback forms exist. But an interactive first-visit tutorial that teaches a cooperative feedback mechanism by having the user physically practice on live content (the LRH Fable), then shows their submission as a sequentially numbered record ("SAVED as 00000129") routed through AI categorization to human domain experts — no prior art.

### Description
First-visit interactive tutorial overlay that appears before any other content. Three guided steps:

**Step 1 — "Like This"**: The existing LRH Fable slideshow plays underneath a semi-transparent overlay. Big instructional text explains: "FEEDBACK REQUESTED — You're exploring a live alpha preview that actually works, but we need YOUR feedback to shape how we continue."

**Step 2 — "Click Item"**: An animated arrow spotlights the Fable slideshow box. The user must CLICK the fable (interactive, not simulated). When they click, the Notes Overlay textbox appears on the fable.

**Step 3 — "Write & Submit"**: The textbox shows placeholder text: "Write what you want in the Textbox that appears, and hit 'Ok' to submit." User writes (or leaves blank), clicks OK. Confirmation shows: "SAVED as 00000129" (sequential 8-digit number from `tour_notes_submitted.submission_number`). The submission feeds into MoneyPenny categorization → Librarian Guild processing.

**Footer**: "Thank you! ☐ Do not show these directions again" — checkbox persists preference.

### Innovation Markers
- First-visit feedback tutorial teaching cooperative contribution by practice
- Sequential 8-digit submission numbering visible to user (makes feedback feel permanent)
- Interactive (user clicks, not simulated) — learn by doing
- Feeds into AI categorization → human Librarian pipeline
- Dismissable with persistent preference

### Formal Claim
A cooperative platform onboarding system comprising an interactive first-visit tutorial overlay that teaches a content feedback mechanism by guiding users through a three-step practice exercise on live content: (1) displaying instructional context over a content preview, (2) directing the user to physically interact with a content element to trigger an annotation interface, and (3) capturing and confirming a user submission with a sequentially numbered identifier that persists in a cooperative review pipeline, wherein submitted feedback is automatically categorized by an AI service and routed to domain-specific human reviewers, and the tutorial includes a persistent dismissal preference so it appears only once per user.
