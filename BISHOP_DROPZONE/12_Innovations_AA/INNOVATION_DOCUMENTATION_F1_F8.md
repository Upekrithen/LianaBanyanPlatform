## F2: CHALLENGE-BASED HIRING

**Innovation Number:** TBD
**Category:** Employment / Workforce
**Priority:** HIGH

### Description
A hiring system where applicants earn credits by completing work samples, reversing the traditional dynamic where job-seeking costs applicants money and time with no guarantee of return.

### Key Features
1. **Earn While Applying** — Applicants complete paid challenges instead of unpaid assessments
2. **Anti-Spam Mechanism** — 1 credit per application prevents mass applications
3. **Free Credit Source** — Order a meal from Let's Make Dinner = 1 free credit
4. **Rejection Compensation** — If rejected, applicant receives 5 credits from hiring entity
5. **Work Sample Portfolio** — Completed challenges become portfolio pieces

### Hiring Flow
```
1. $5 membership (one-click)
2. Complete Keirsey Temperament Sorter (required)
3. Browse available challenges
4. Spend 1 credit to apply
5. Complete challenge (earn payment regardless of hiring decision)
6. If hired: onboard
7. If rejected: receive 5 credits + keep challenge payment
```

### Challenge Types
- **Straight Pay** — Fixed payment for completed work
- **Contest Model** — Winner-take-all with runner-up compensation

### Patent Claims (Suggested)
1. Method for compensating job applicants during application process
2. System for converting work samples into portfolio credentials
3. Anti-spam mechanism using economic friction (credit cost)
4. Process for guaranteeing applicant compensation regardless of hiring outcome

## F3: MILESTONE-BASED EMPLOYMENT

**Innovation Number:** TBD
**Category:** Employment / Contracts
**Priority:** HIGH

### Description
Employment contracts divided into 10 segments, allowing either party to pivot at natural breakpoints without the binary "employed/fired" dynamic.

### Key Features
1. **10-Segment Structure** — Each contract has 10 defined milestones
2. **Mutual Pivot Rights** — Either party can exit at any milestone
3. **Prorated Compensation** — Clear payment for completed segments
4. **Skills Verification** — Each segment completion creates verifiable credential
5. **Reputation Building** — Successful segments build worker reputation score

### Contract Flow
```
Contract Start
├── Milestone 1 (10%) — Review point
├── Milestone 2 (20%) — Review point
├── ...
├── Milestone 9 (90%) — Review point
└── Milestone 10 (100%) — Completion

At each milestone:
- Worker can pivot to different project
- Employer can reassign worker
- Both parties have full visibility of next milestone requirements
```

### Patent Claims (Suggested)
1. Employment contract structure with defined mutual exit points
2. System for prorating compensation across contract milestones
3. Method for building verifiable credentials from partial contract completion
4. Process for matching workers to projects based on milestone completion history

## F6: VIRTUAL EXPERTS — AI NOIDS AS TOOLS

**Innovation Number:** TBD
**Category:** AI / Assistance
**Priority:** MEDIUM

### Description
AI assistants positioned explicitly as tools (like wheelbarrows, oars, or scalpels) rather than autonomous agents, maintaining clear human accountability.

### Core Philosophy
> "AI noids are tools — wheelbarrows, oars, scalpels. The human using the tool is responsible for the outcome."

### Key Features
1. **Tool Framing** — AI presented as capability amplifier, not decision-maker
2. **Human Accountability** — All AI actions attributed to human operator
3. **Audit Trail** — Complete logging of human-AI interactions
4. **Capability Limits** — Clear boundaries on what AI can and cannot do
5. **Override Priority** — Human input always supersedes AI suggestion

### Implementation
- **Wheelbarrow Mode** — AI handles heavy lifting (data processing, research)
- **Oar Mode** — AI provides direction assistance (navigation, recommendations)
- **Scalpel Mode** — AI enables precision (detailed analysis, specific tasks)

### Patent Claims (Suggested)
1. AI assistance system with explicit tool-based framing
2. Method for maintaining human accountability for AI-assisted actions
3. Architecture for tiered AI capability based on task type

## F8: REPUTATION QUANTIFICATION

**Innovation Number:** TBD
**Category:** Trust / Governance
**Priority:** MEDIUM

### Description
A system for quantifying reputation through "risk points" on projects, with fair governance mechanisms preventing abuse.

### Key Features
1. **Risk Points** — Measurable reputation currency earned/lost through project outcomes
2. **Contextual Weighting** — Points weighted by project difficulty and domain
3. **Decay Function** — Old incidents matter less than recent performance
4. **Appeal Process** — Disputed point assignments can be appealed
5. **Transparency** — All point calculations visible to affected party

### Governance Protections
- **Cap on Single-Incident Loss** — No single failure can destroy reputation
- **Rehabilitation Path** — Clear process for recovering from setbacks
- **Peer Review** — Point assignments reviewed by domain experts
- **Anti-Gaming Measures** — Prevents artificial reputation inflation

### Patent Claims (Suggested)
1. Reputation quantification system with contextual weighting
2. Method for time-based decay of reputation events
3. Governance system for fair reputation dispute resolution
4. Anti-gaming architecture for reputation systems
