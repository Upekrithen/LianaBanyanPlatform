### #2273 (candidate) — Fingerprint-Based Incremental Index Reconciliation
- **Crown Jewel:** Candidate
- **Category:** AI-infrastructure / reliability
- **Session of Origin:** B115 (K429 dispatched) / B116 (K429 shipped, verified)
- **Provenance:** K429 implementation by Knight; pain-signal origin B115 ("MCP is unavailable").
- **A&A Formal:** Not yet drafted (implementation exists: `librarian-mcp/src/indexer/fingerprint.ts`)
- **Description:** SHA-256 tree-hash fingerprint over file mtimes across a declared set of scan directories (in this implementation: 12 directories, 7,829 files).
