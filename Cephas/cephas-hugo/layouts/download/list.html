▶ RUNG 2 Wrasse Sub-millisecond Pre-injection + Sub-ms delivery  ·  0.059ms mean

Standard AI memory systems wait until you ask — then go look. Mnemosyne pre-loads what you need before the AI starts answering. That's the difference between a search engine and an expert who already knows your project.

[Founder expands Skipping Stones copy — contrast vs RAG in plain terms]

Canon-routed retrieval that loads the relevant canonical entity into agent context BEFORE the reasoning step. Not embedding-similarity RAG. Not agent-directed memory. Wrasse fires on trigger-phrase match, routes to canonical Eblet, and pre-injects — the agent begins reasoning with ground truth already present. Empirical anchor: 0.059ms mean lookup per Prov 15 #2332.

Rung 2 is the active mechanism that USES the ROM tier (Rung 1). Without canonical permanence, Wrasse would retrieve mutable approximations. With Rung 1: canonical permanence × sub-ms delivery = ground-truth-at-reasoning-time.

RAG vs Wrasse: RAG retrieves after the agent encounters uncertainty — embedding-similarity, approximate, potentially stale. Wrasse pre-injects before the reasoning step on trigger-phrase match — canonical-routed, immutable, structurally not agent-directed. The "pre-" in pre-injection is load-bearing: the agent receives ground truth at context-load time, not at uncertainty-signal time.

Cross-references: Prov 15 #2332 · Stone-Tablet canon · Wrasse architectural spec.

Wrasse implements a two-phase lookup: Phase 0 = pheromone-strength fast-path (in-process IndexState hash lookup, 0.059ms mean) → Phase 1 = RPC fallback to Librarian MCP (full Cathedral scan, ≤20ms typical). Phase 0 accounts for >96% of hits in production (Detective empirical: 49:1 hit ratio vs filesystem grep at B134). The pre-injection pipeline: trigger-phrase tokenized at context-load time → Wrasse fires → canonical Eblet loaded into system prompt prefix → agent reasoning begins with ground truth in-context. This is structurally different from agent-directed retrieval: the agent does not issue a memory query; the substrate delivers without agent request.

Empirical receipt: Prov 15 claim #2332 — 0.059ms mean across 1,000 lookup samples. Phase-0 hit: O(1) hash lookup on IndexState.pheromone_index. Phase-1 hit: full-text scan bounded by Cathedral Scribe size (KnightQueue.jsonl ≤ 400KB typical).

Filing anchor: Prov 20 covers Wrasse trigger-phrase-matched pre-injection as a substrate primitive distinct from RAG, agent-directed retrieval, and user-initiated memory query.

▶ RUNG 3 Pheromone Strength + Half-life Decay + Substrate-driven priority

The more you use something, the easier it is to find next time. Mnemosyne tracks which memory fragments you reach for most and puts them closer to the front. Like a well-worn path — the trail gets clearer with use, not murkier.

[Founder expands Skipping Stones copy — pheromone as trail metaphor for first-timers]

Each canonical entity carries a pheromone strength field subject to configurable half-life decay. This enables substrate-driven prioritization of canonical entity pre-injection weight — without agent-directed priority signals. The substrate "smells" of recent relevance: Eblets accessed or reinforced recently carry higher pheromone strength and are weighted more heavily in the pre-injection stack.

Rung 3 composes with Rungs 1–2 as a prioritization layer. Pheromone strength × Wrasse pre-injection × canonical permanence = substrate-driven prioritized delivery of immutable ground truth.

Configurable half-life: the decay schedule is substrate-configurable, not agent-configurable. Agents cannot boost their own priority signals. The decay reflects actual usage patterns across the cooperative, not agent assertions about importance.

Prior-art carve-out: no prior-art system uses substrate-usage-pattern-driven pheromone decay as a canonical entity prioritization primitive. Competitor systems allow agents to flag memory items — that is agent-directed priority. Liana Banyan's model trusts substrate usage patterns.

Cross-references: Prov 15 (pheromone empirical anchors) · swarm-intelligence prior-art carve-out in substrate canon.

Pheromone decay is implemented as a 3-class hybrid in librarian-mcp/src/scribes/pheromone.ts (BP046B SAGA 16): anchor (no decay — canonical-tier permanence), linked (decay deferred to most-recent sibling timestamp across linked_ids set), transient (standard exponential half-life decay). The decay score function: S(t) = S₀ × 0.5^((t − t₀) / t½) for transient class; anchors return S₀ unconditionally; linked returns max(S_self, max(S_sibling)) at query time. This 3-class system is the live SAGA 16 implementation, committed at a5dac46.

IndexState coupling: decayScore() accepts IndexState at query time to perform linked-sibling lookups without serializing the full index. This is an O(|linked_ids|) scan bounded by the linked set size, typically ≤ 12 siblings.

Filing anchor: Prov 15 covers 3-class decay as a substrate prioritization primitive. Transient class maps to standard swarm-pheromone prior art (PSO, ACO). Anchor + linked classes are novel composites — no prior art uses substrate-usage-driven decay with explicit permanence bypass at the canonical tier.

Patent cross-reference: Prov 15 (3-class pheromone) · BP046B SAGA 16 commit a5dac46 · pheromone.ts lines ~80–140.
