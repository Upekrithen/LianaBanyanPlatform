## Security Definer Functions (2 created)

Both functions lock `search_path = public, pg_catalog` per spec.

### 1. `public.mark_red_carpet_grant_used(p_grant_token text)`
- **Returns:** void
- **Purpose:** Allows anon/just-authed users to mark their red_carpet_access grant used + set recipient_user_id = auth.uid()
- **Logic:**
  ```sql
  UPDATE red_carpet_access
  SET used_at = now(), recipient_user_id = auth.uid()
  WHERE grant_token = p_grant_token
    AND used_at IS NULL
    AND grant_expires_at > now()
  ```
- **Security:** SECURITY DEFINER bypasses RLS (recipient_user_id is NULL at call time)

### 2. `public.record_cue_card_click(p_card_id uuid, p_anon_session_id text, p_ip_country text, p_ua_class text)`
- **Returns:** uuid (click_token)
- **Purpose:** Allows anonymous visitors to record clicks without auth
- **Logic:**
  ```sql
  INSERT INTO cue_card_share_clicks (cue_card_id, anonymous_session_id, ip_country, user_agent_class)
  VALUES (p_card_id, p_anon_session_id, p_ip_country, p_ua_class)
  RETURNING click_token
  ```
- **Security:** SECURITY DEFINER bypasses RLS (table is service_role-only for privacy)
