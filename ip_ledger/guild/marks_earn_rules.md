## BLOOD RULE: No New Rates Without Founder Ratification

All Marks earn rates below are Founder-ratified per BP086 MARKS CLEARING TABLE.
No agent (Knight, Bishop, Rook, Pawn) may introduce new rates or modify existing
rates without explicit Founder sign-off. New rate proposals go to Founder review
before any implementation.

---

## Guild Developer Marks Earn Table

| Activity | Rate | Notes |
|---|---|---|
| Work pool completion | 10% of allocated work value | Applied at work close; proportional to peer contribution |
| Peer code review (verified) | 0.05 Marks | Must be verified by at least one other Guild member |
| Bounty completion | Proportional to bounty size | Per individual bounty specification |
| Documentation contribution | 0.05 Marks | Per accepted doc; steward-verified |

---

## General Platform Marks Rules (applicable to all Guild members)

- Marks are cooperative participation units -- NOT securities, NOT equity
- Marks do not represent an ownership stake or financial return
- Use: "participation", "allocation", "contribution", "back"
- Do NOT use: "equity", "shares", "dividends", "ROI", "invest"

---

## Creator Pay-Through

When a Guild developer's application generates revenue through the platform:
- Creator/worker receives **83.3%** (never round to 83%; never 84%)
- Platform margin: **Cost + 20%**
- Example: on a $500 transaction, the creator receives $416.67

---

## Membership Requirement

Active Guild membership requires a valid $5/year LB cooperative membership.
Marks accumulation pauses if membership lapses; resumes upon renewal.

---

## Rate Change Process

1. Developer proposes new rate in writing to Guild steward
2. Steward escalates to Founder review queue
3. Founder ratifies or rejects
4. Ratified rates are added to this file with session stamp
5. No implementation until this file is updated and committed
